Environment Variables

Variables are quantities on memory that can store values so that we can refer to them later by the name of that variable.

On a shell, to create a variable, a variable name must be specified followed by an equals sign without any space in the middle.
Variables might be set empty or the equals sign is followed by a value.

Note that there are no spaces between variable name, equals operator, and value.
Values can be double-quoted to represent a string which might contain a space.
Beware that by eliminating double-quotes, spaces will cut value in pieces and nothing after first space will be stored in that variable.

To access the value stored in a variable use $ before variable’s name without any space in the middle.

Here we use echo command to print the value stored in variables.
Environment variables are heavily used in shell configuration. For example PATH variable is used to tell the shell where it can find the external commands like mkdir, which we discussed before.

PATH holds a list of all directories which the shell will seek for the commands you type, delimited by colon (:).
That is, when you type a command like mkdir on your shell and hit enter, shell will look for a program named mkdir in specified directories respectively.

As the type command indicated, shell will find the mkdir program in /usr/bin directory, so it will stop searching after the first match.

A list of environment variables used by shell can be found by env command.
Don’t fool around with these variables, DO NOT change any of them.

Two or more commands might have the same name on a system, or one com- mand might be stored in more than one directory, then shell will execute the first matched directory in the PATH environment variable.

Finally, which command, tells you where external commands are and by giving -a option it indicates how many instances of a command exists in PATH directories.
In an example, the instance of mkdir in /usr/bin will be executed because the /usr/bin directory preceeds the /bin directory, though they both contain the mkdir command.

    #!/usr/bin/env bash
    
    # store a string
    project="Linux Administration"
    
    # only Jane will be stored
    name=Jane Doe
    
    # print variable
    echo "$name"
    
    # PATH environment variable
    echo "$PATH"
    
    # list of all environment variables
    env
