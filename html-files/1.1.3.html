Essential Commands

We need a few basic commands before we can further understand shell features.

Print Working Directory
As the name suggests, pwd tells you where you are in the Linux directory hierarchy.
When you log into a shell, you directly jump into your home directory.
Home directory is also known as Ìƒ character, normally set to /home/user directory where user is the name of the user you are logged in to.
dot . indicates current directory and .. indicates parent directory.

List Files and Directories
ls command lists the entry of current working directory if no paramters are given, otherwise it will print the entry of the path you gave it to.

Change Directory
cd command is straightforward, it will change your directory to whatever path you give it to as its paramter.
Of course, the directory must exists or it will complain that no such file or directory exists.

Create File
touch is used to create files.
You can give it multiple file names as paramters and it will create all of those files.

Make Directory
mkdir command makes one or as many directories as you specify in front of it as parameters.

Removing File and Directories
rm command removes files.
To remove directories, rmdir command might be used only if the directory is empty, otherwise, rm -r command with its option must be specified.
-r option indicates --recursive which basically means rm cannot remove many files in one time, but it recursively searches for files in given directory and remove them one by one.

Echo Back Something
echo command might sound irrational at first sight, but it will echo back whatever you give it to as parameters.
Soon in later chapters we will see that in Bash we can create variables.
To print the value within a variables, we shall use echo as it echoes back what that variable holds.

Display File Content
cat command prints the contents of one or as many files as we specify as its paramters.
Printing all of the contents of a big text file might not be a good idea.
Later, we will practice commands which gives of enough control to print just a part of a file which we are looking for.
Of course, printing the contents of a non-printable file, e.g. binary files,
will result in dumping garbage characters on the terminal's output.

Copy File or Directory
cp requires source files and the destination path in order to perform a copy of source files into destination.
To copy directories, -r option must be given in order to recursively copy all the files within a directory.

Move File or Directory
mv command moves files or directories as source into destinaion.
Move command will copy files then eliminates source files.

Rename File or Directory
On Linux, we don't have a rename command. Instead, what we do here is to move files in source path, e.g. current directory, using mv command, and move them into destination path, e.g. also current directory, with a new name.

Source

essentials.sh 

    #!/usr/bin/env bash
    
    # print working directory
    pwd
    
    # list entry
    ls /usr/local/
    
    # change directory
    cd .
    
    # make directory
    mkdir /tmp/directory
    
    # create file
    touch /tmp/file
    
    # remove file or directory
    rm /tmp/file
    rm --recursive /tmp/directory
    
    # echo back and write to file
    echo "message" > /tmp/sample
    
    # print contents of a file
    cat /tmp/sample
