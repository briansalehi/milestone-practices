Sed

Sed is a stream processing tool which has its own reputation and we will not cover everything about it (we can't) in a single practice.

To learn sed in depth, visit stream processing chapter.

We will cover the most basic and regularly used features of Sed here, which are deleting and printing lines, and also substitution of strings.

Using Sed is straightforward, you call sed following options and then Sed script surrounded by single quotes.
The Sed script itself follows a specific syntax, first the address of lines can be given by numbers or string pattern, then a command is followed by that address to give desired results.


sed options '/address/command'

In case of addess, we are using line numbers in this example.
And for commands, we used d to delete mentioned lines.
We used p to print specified lines, which requires -n to avoid duplicate printing of lines, simply because sed itself also prints lines alongside our print command.
And then we used s to substitute word amazing to useful.

Source

sed.sh 

    #!/usr/bin/env bash
    
    cat << EOF > /tmp/sample
    You can not belive how amazing Sed can be
    until you used it!
    EOF
    
    # deleting line
    sed '2d' /tmp/sample
    
    # printing line
    sed -n '1p' /tmp/sample
    
    # substitution of string
    sed 's/amazing/useful/' /tmp/sample

Execution

run


bash ./sed.sh

Output


You can not belive how amazing Sed can be
You can not belive how amazing Sed can be
You can not belive how useful Sed can be
until you used it!
